Usage

Drop-in cat replacement

mcat file.txt
mcat -n file.txt
mcat -A file.txt
echo "hello" | mcat

Structured data

mcat data.parquet
mcat data.csv
mcat --format jsonl data.parquet
mcat --schema data.parquet

Filter & slice

mcat --head 10 data.parquet
mcat --tail 5 data.csv
mcat --columns name,age data.parquet
mcat --count data.parquet

Column statistics

mcat --stats data.parquet
mcat --stats --columns age,salary data.parquet
mcat --stats data.csv.bz2

Compressed files

mcat data.parquet.gz
mcat data.csv.bz2
mcat data.jsonl.zst
mcat s3://bucket/logs.jsonl.gz --head 100

Remote sources

mcat s3://bucket/data.parquet
mcat gs://bucket/data.parquet
mcat https://example.com/data.csv
mcat --s3-endpoint https://play.min.io \
  s3://mybucket/data.parquet

Remote Sources

mcat uses fsspec under the hood for seamless remote file access. No full downloads — streaming and range requests are used where possible.

ProtocolWorks viaNotes
s3://s3fs + boto3AWS S3, MinIO, R2, B2, Spaces
gs://gcsfsNative GCS — richest feature set
https://fsspec built-inRange requests where supported
Any S3-compats3fs + --s3-endpointMinIO, Cloudflare R2, Backblaze B2, DO Spaces

Authentication

# Option 1: AWS CLI (recommended)
aws configure   # stores in ~/.aws/credentials
mcat s3://my-bucket/data.parquet

# Option 2: Environment variables
export AWS_ACCESS_KEY_ID=AKIAIOSFODNN7EXAMPLE
export AWS_SECRET_ACCESS_KEY=wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
export AWS_DEFAULT_REGION=us-east-1
mcat s3://my-bucket/data.parquet

# Option 3: Named profile
AWS_PROFILE=myprofile mcat s3://my-bucket/data.parquet

# Option 4: Temporary session token (MFA, assume-role)
export AWS_SESSION_TOKEN=AQoXnyc4lcK4w4OIAHPOjgWDSB...
mcat s3://my-bucket/data.parquet
# Via --s3-endpoint flag
mcat --s3-endpoint https://play.min.io s3://mybucket/data.parquet

# Via environment variable
export FSSPEC_S3_ENDPOINT_URL=https://play.min.io
mcat s3://mybucket/data.parquet

# Cloudflare R2 example
mcat --s3-endpoint https://<account-id>.r2.cloudflarestorage.com s3://bucket/file.parquet

# MinIO with credentials
AWS_ACCESS_KEY_ID=minioadmin AWS_SECRET_ACCESS_KEY=minioadmin \
  mcat --s3-endpoint http://localhost:9000 s3://mybucket/data.parquet
# Option 1: gcloud CLI (recommended)
gcloud auth application-default login
mcat gs://my-bucket/data.parquet

# Option 2: Service account key
export GOOGLE_APPLICATION_CREDENTIALS=/path/to/service-account.json
mcat gs://my-bucket/data.parquet
